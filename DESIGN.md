# 设计文档：直播间产品图片生成 Agent

## 1. 产品概述

### 1.1 目标

为电商直播运营团队提供一个自动化工具，根据参考直播间照片和产品图，快速生成不同主播、不同背景下的直播间产品展示图片，用于营销素材制作。

### 1.2 核心价值

- **效率**：替代人工 PS 合成，从数小时缩短到数分钟
- **一致性**：通过自动评估保证产出质量
- **灵活性**：支持不同人物、背景、商品位置的组合

### 1.3 目标用户

- 电商直播运营人员
- 跨境电商营销团队
- 内容制作外包团队

## 2. 系统架构

### 2.1 整体流程

```
用户输入（参考图 + 产品图 + 需求）
        ↓
   初始图片生成（Gemini API）
        ↓
   自动质量评估（5维度打分）
        ↓
   通过? → 输出最终结果
   未通过 → 生成修正指令 → 继续对话修正
        ↓
   分数下降? → 回滚到最佳状态重新生成
        ↓
   达到最大轮次 → 输出最佳结果
```

### 2.2 核心模块

| 模块 | 职责 |
|------|------|
| Agent (agent.py) | 编排迭代循环、回滚控制、进度上报 |
| Generator (generator.py) | Gemini API 通信、多轮对话状态管理 |
| Evaluator (evaluator.py) | 图片质量评估、结构化评分 |
| FixStrategy (fix_strategy.py) | 根据评估结果生成针对性修正指令 |
| PromptBuilder (prompt_builder.py) | 组装初始生成 prompt |
| Web (web/) | HTTP API + SSE 实时推送 + 前端界面 |

### 2.3 API 调用方式

通过中转代理调用 Gemini REST API：
- Base URL: `https://api.vectorengine.ai/v1beta/models/`
- Model: `gemini-3.1-flash-image`
- 认证: `x-goog-api-key` header

两种调用模式：
- **图片生成**：responseModalities=["image", "text"]，temperature=1.0
- **视觉评估**：responseModalities=["text"]，temperature=0.2

## 3. 迭代策略设计

### 3.1 多轮对话修正

利用 Gemini 的多轮对话能力，在同一个 conversation 中累积修正：
- 第一轮发送完整参考图 + 产品图 + prompt
- 后续轮次只发送文字修正指令（Gemini 限制不能在后续轮次重发图片）
- 模型能看到完整对话历史，基于上下文修正

### 3.2 分数回滚机制

**问题**：多轮对话中负面描述累积，模型可能越改越差

**解决方案**：
- 追踪每轮评分和最佳状态
- 当分数比上轮下降 > 0.3 时，触发回滚
- 丢弃当前对话，回到最佳状态重新开始
- 避免在错误方向上越走越远

### 3.3 提前终止

- **严格通过**：所有5个维度 >= 4 分
- **Good enough**：总分 >= 4.5 且所有维度 >= 3 分
- 满足任一条件即终止，避免过度迭代

### 3.4 Fresh Restart

连续 N 轮（默认5）未通过时，完全丢弃对话历史，用原始 prompt 重新生成。

## 4. 评估体系

### 4.1 评估维度（每项 1-5 分）

| 维度 | 评估内容 |
|------|---------|
| 姿势真实感 | 是否参考原图、是否自然不对称 |
| 产品准确度 | 前景产品文字/logo/外观是否匹配（背景货架不扣分） |
| 产品大小 | 相对人物比例是否合理 |
| 需求匹配度 | 人物、背景、商品位置是否符合 |
| 物理合理性 | 无悬浮/穿模、无平台UI元素 |

### 4.2 评估独立性

- 每次评估是独立的单轮 API 调用
- 不携带之前轮次的评估历史
- 同时发送：生成图 + 参考图 + 产品图进行三方对比
- 评估 prompt 包含完整生成约束条件

## 5. Prompt 工程

### 5.1 关键约束

- 前景产品严格匹配，背景装饰放宽
- 姿势参考原图角度，避免正面对称
- 不允许幻想产品的其他形态（内包装等）
- 不允许平台 UI 元素

### 5.2 动态内容

- 用户需求描述（人物/背景/商品位置/补充说明）
- 目标图片尺寸（根据比例选项计算）
- 修正轮次中的具体问题和修正指令

## 6. Web UI 设计

### 6.1 输入表单

- 参考图上传（单张）
- 产品图逐张添加（支持删除）
- 人物描述（文本）
- 背景描述（预设 + 自定义）
- 商品位置（桌前/手持/参考原图/自定义）
- 图片比例选择
- 迭代轮次选择
- 补充说明

### 6.2 实时进度

- SSE 推送每个事件（生成/评估/修正/回滚/完成）
- 每轮结果展示为独立卡片：图片 + 评分 + 问题列表
- 提示词查看按钮
- 回滚/重启时显示原因提示

### 6.3 结果展示

- 最终图片大图展示（点击放大）
- 所有轮次图片可下载
- 二次编辑功能（文字指令微调）

## 7. 已知局限与后续规划

### 7.1 当前局限

- **产品文字保真**：Gemini 对产品上的文字复现能力有限，尤其是小字和复杂 logo
- **评估稳定性**：同一张图多次评估分数可能有波动
- **API 限流**：中转 API 有频率限制，密集调用时需等待

### 7.2 后续规划

- OpenCV 产品合成方案：先生成场景，再用图像处理将真实产品合成上去
- 批量生成模式：一次生成多个不同方案
- 人物模板库：预设不同风格的主播形象
- 历史任务管理：保存和对比不同任务的结果
